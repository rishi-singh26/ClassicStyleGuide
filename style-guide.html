Design tokens

All visual values are defined as CSS custom properties on :root. Dark mode overrides the same variables inside @media (prefers-color-scheme: dark). Never hardcode colors — always reference a token.

backgrounds — 4 levels of depth
--bgpage base
--bg2sidebar, cards
--bg3hover states
--bg4inputs, badges
text hierarchy — 3 levels
--textPrimary text — headings, labels, code
--text-mutedBody text — descriptions, notes, content
--text-dimDe-emphasized — labels, hints, timestamps
accent
--accentBorders, active states, buttons
--accent-lightText on dark, highlighted words in headings
--accent-dimActive nav background, icon backgrounds (semi-transparent)
borders
--borderDefault border — all cards, dividers, inputs
--border-hoverBorder on hover — slightly more opaque
When creating a new page, copy the entire :root { } and @media (prefers-color-scheme: dark) { :root { } } blocks verbatim. Do not add intermediate variables or rename tokens — the component classes depend on these exact names.

Max-width
The .main column has max-width: 860px and uses Bootstrap's flex-grow-1. The sidebar is 220px wide with Bootstrap's flex-shrink-0. On screens narrower than 768px, the sidebar becomes a slide-in drawer and the main area uses padding: 0 1rem 4rem.

Sidebar & navigation

The sidebar is sticky, full-height, and scrollable independently. Navigation links highlight as the user scrolls using a JS scroll listener.

nav link states
Default — var(--text-muted), no border Hover — var(--text), bg3 background Active — accent-light text, accent left border, accent-dim bg
JS — scroll-activated nav highlight
const allSections = document.querySelectorAll('section.section, .page-header');
const allNavLinks = document.querySelectorAll('.nav-link');

function navTo(id) {
  const el = document.getElementById(id);
  if (el) el.scrollIntoView({ behavior: 'smooth', block: 'start' });
}

window.addEventListener('scroll', () => {
  let cur = '';
  allSections.forEach(s => {
    if (window.scrollY >= s.offsetTop - 120) cur = s.id;
  });
  allNavLinks.forEach(l => {
    const target = l.getAttribute('onclick')?.match(/'([^']+)'/)?.[1];
    l.classList.toggle('active', target === cur);
  });
}, { passive: true });
Never name the scroll function scrollTo — this conflicts with the native window.scrollTo(x, y) browser API and silently breaks navigation. Always use a distinct name like navTo.

JavaScript patterns

Three small JS functions cover all interactivity. Copy them verbatim into every page's <script> block.

JS — all three functions
/* ── 1. Sidebar scroll-tracking ───────────────────────── */
const allSections = document.querySelectorAll('section.section, .page-header');
const allNavLinks = document.querySelectorAll('.nav-link');

function navTo(id) {
  const el = document.getElementById(id);
  if (el) el.scrollIntoView({ behavior: 'smooth', block: 'start' });
}

window.addEventListener('scroll', () => {
  let cur = '';
  allSections.forEach(s => {
    if (window.scrollY >= s.offsetTop - 120) cur = s.id;
  });
  allNavLinks.forEach(l => {
    const target = l.getAttribute('onclick')?.match(/'([^']+)'/)?.[1];
    l.classList.toggle('active', target === cur);
  });
}, { passive: true });

/* ── 2. Toggle expandable blocks (step-block, cmd-card) ─ */
function tog(el)    { el.classList.toggle('open'); }   // step-block: onclick="tog(this)"
function toggle(el) { el.classList.toggle('open'); }   // cmd-card:   onclick="toggle(this)"

/* ── 3. Copy button for code blocks ─────────────────────── */
function cp(btn) {
  const text = btn.closest('.code-block').querySelector('pre').innerText;
  navigator.clipboard.writeText(text).then(() => {
    btn.textContent = 'copied!';
    btn.classList.add('copied');
    setTimeout(() => { btn.textContent = 'copy'; btn.classList.remove('copied'); }, 1800);
  });
}
Critical: Use navTo, not scrollTo. The browser's native window.scrollTo(x, y) silently overrides any custom function named scrollTo, breaking all sidebar navigation clicks.

Rules to always follow
1. Every color must come from a CSS variable — never hardcode hex values.
2. Use --serif only for h1 and h2, always italic, always weight 300.
3. Use --mono for everything else — body, labels, code, buttons.
4. The em tag inside headings renders non-italic in --accent-light — this is the signature typographic move of the style.
5. Never name the nav function scrollTo.
6. Sections must have an id that matches the navTo() argument in the sidebar links.
7. .page-header must have id="overview" (or whatever the first nav link targets).
